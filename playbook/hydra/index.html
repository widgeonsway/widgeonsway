Hydra

Hydra is a popular and powerful password-cracking tool used for brute-forcing login credentials on various network services. It supports a wide range of protocols, such as HTTP, FTP, SSH, RDP

Basic Facts:

  • A brute forcing tool that performs password cracking
  • Can perform thousands of login attempts per minute.
  • Use -L when you are using a wordlist or a text file to guess the username
  • Use -l when you already have the username
  • Use -P when you are using a wordlist or text file to guess the password
  • Use -p when you already know the password and are guessing the username
  • Use -V will show all login/password attempts
  • Use -o <filename> to save all results into a file

Common Scans:

  • hydra -L <username-wordlist> -P <password-wordlist> <IPAddress> port
  • hydra -l <username> -P <password-wordlist> <IPAddress> port
  • hydra -L <username> -p <password> <IPAddress> port

CTF Usage:

1. Using hydra may be against the rules of certain CTF's. It also takes out some of the learning and fun you would have finding the credentials in the intended way.
2. If you are sure you are allowed to use hydra, make sure you know the target IP and which service you will be attacking.
3. Always try common default credentials first. Even if they don't work, you may get an error message such as "invalid password". This means that the username you tried does exist and now you have a starting point!
4. You may find a username right away that you want to try. You would do this with -l
5. Sometimes there are multiple users and you will be unsure of which one is the correct one to proceed with to get user.txt, so you can put all potential users into a text file and then use -L to use that file.
6. Be aware that not all usernames/passwords will be in the wordlists you choose to use, so either Hydra will work or it won't and its time to move on to something else.

1. POST or GET?

    You need to determine what type of request you are dealing with.
  • If it is a log in page, you are sending a POST request.
  • You can also verify this by right clicking on the webpage and clicking inspect and go to the network tab. Watch when you enter the password.
  • You can also intercept the request with Burp Suite.
