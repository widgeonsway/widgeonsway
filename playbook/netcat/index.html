NetCat

A common favorite tool for reverse shells

Basic Command:

    While there is a lot of ways to use NetCat, the most common form is:
  • nc -lnvp <port>

These flags (-l -n -v -p) are the "catch all" for Netcat. This puts NC into listen mode (-l), ignores DNS and only takes in IP's(-n), focuses on your specified port (-p) and gives verbose- or a lot of output(-v). You can use them in any order or alone.

CTF Usage:

During a CTF you would use netcat when you have a payload ready to receive. For example, if the CTF challenge website has somewhere you can upload files, you can try to first see what type of files it accepts and see if you can get it to execute. Reverse shells are one of the main ways you will interact with a target computer. Netcat will be the main tool you use to "catch" a shell.
The most common usage is the reverse shell listener.
To set this up you type: nc -lnvp port
This will open up that specific port and wait for a connection

You can also use nc to transfer files by adding "filename"
This will send the file to someone who connects on port 4444
