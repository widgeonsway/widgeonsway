Ports

This is a list of commonly used ports in CTF challenges and what to do when you see them. Keep in mind, CTF makers may change which service is running on which port.

Port 21: FTP (File Transfer Protocol)

  • If you ran your initial nmap scan with no tags, you can run another nmap scan "nmap -sC -sV -p21 <IPAddress>"
  • Try logging into the ftp. Some options for the username:password are anonymous:anonymous, ftp:ftp, anonymous: (no password)
  • hydra -L users.txt -P passwords.txt ftp://<IPAddress>
  • hydra -V -l widgeon -P /usr/share/wordlists/rockyou.txt ftp://<IPAddress>
  • This is a useful script to enumerate users ftp-user-enum

Port 22: SSH (Secure Shell)

  • hydra -l <username> -P <wordlist> <IPAddress> ssh
  • hydra -L users.txt -P <wordlist> ssh://<IPAddress>
  • ssh-keygen will help you generate new keys. Keys are typically found in /home/<USER>/.ssh

Port 23: Telnet

  • telnet <IPAddress> 23
  • nc -v <IPAddress> 23
  • Use WireShark

Port 25: SMTP (Simple Mail Transfer Protocol)

  • Identify software and version number so you can find version specific exploits

Port 53: DNS (Domain Name System)

  • gobuster dns -d <http.website.com> -w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt

Port 80: HTTP (Hypertext Transfer Protocol)

  • Inspect the webpage with Wappalyzer
  • hydra -l username -P </usr/share/wordlists/rockyou.txt> http://<IPAddress>:80
  • hydra -l <username> -P <wordlist> <IPAddress> <HTTP Method> "<Path to Login Page>:<Request Body>:<Failed Attempts>
  • ffuf -u <http.website.com>/indexFUZZ -w /usr/share/seclists/Discovery/Web-Content/web-extensions.txt

Port 143: IMAP

  • hydra -l <username> -P <wordlist> <IPAddress> -s 143 imap -V
  • Identify software and version number so you can find version specific exploits
  • nikto -h <IPAddress> -p 143

Port 139/445: SMB (Server Message Block)

  • Identify software and version number so you can find version specific exploits

Port 443: HTTPS (HTTP Secure)

  • nikto -h website.com -ssl

Port 1433: MSSQL (Microsoft SQL Server)

  • Identify the version of MSSQL and look for vulnerabilities specific to it that will work on the system.
  • Try to log in with default credentials
  • SQLmap

Port 3306: MySQL Database

  • Identify the version of MSSQL and look for vulnerabilities specific to it that will work on the system.
  • Try to log in with default credentials
  • SQLmap

Port 3389: RDP (Remote Desktop Protocol)

  • Log in using xfreerdp. This will give you GUI access
  • Attempt to log in with default credentials
  • hydra -l <username> -P <wordlist> rdp://<IPAddress>

Port 8080: Alternative HTTP Port

  • Go to the IP:8080 to check it out
  • Start a Gobuster or FFUF scan. These are the same scans as for port 80
