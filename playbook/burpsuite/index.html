Burp Suite

Burp Suite comes pre installed on Kali. You can download it here if you are using another distro Burp Suite can be quite intimidating to use at first, it has a lot going on. It is a web application security testing tool.

Starting Burp:

Step 1:

Launch Burp Suite and decide what type of project this will be. You can either create a new file, open an existing file or create a temporary file. It's easiest to just check temporary and then hit the next button.

This will bring you to another screen that has the options to "Use Burp Defaults", "Use setting saved with Projects" and "Load from Configuration File." Let's choose "Use Burp Defaults" and then click Start Burp. That will bring you to this screen:

How to Use:

Target

This tab lets you view the request and response history. Add settings to Proxy so it only intercepts in relation to the target

Proxy: Allows you to intercept and modify web traffic

  • 1. Make sure that the Intercept button is off. This is to make sure that all previous connections are finished resolving and that when you turn intercept back on, you are only working with the requests you want.
  • 2. Open the webpage you want to work with in Burp Suite. This is a new feature that is really nice and makes Burp Suite more enjoyable to use.
  • 3. Turn intercept on and send the request through. These means, enter your log in attempt, upload/download your file, anything that will generate a POST or GET request.

Intruder

Intruder can be used for fuzzing, IDOR, brute-forcing and more.

  • 1. First you want to identify where to place the payload. Click "Clear §" and then "Add §" for that location. For example the field username could be attacked: username=§PAYLOAD§
  • 2. Load up your wordlist. On the payloads tab you should be able to upload a wordlist to use as your payload. This will send out requests to the web server with each item on your list and give you a response for each.
  • 3. Start Attack. You will be given a response code and length for all your results, this will involve some analysis.

Repeater

Once you have found your request that you want to work with, it's a good idea to send it to repeater. This tab allows you to send over a specific request to the host very quickly and easily. It also allows you to modify any of the fields, which make it a great place to craft certain exploits. You can also right click and send the request as a curl command which will put the command into your clipboard. Or you can save the request to a file.
